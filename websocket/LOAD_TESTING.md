## 📊 Мониторинг сервера

### Вариант 1: Скрипт мониторинга (рекомендуется)

```bash
./monitor_server.sh
```

Показывает в реальном времени:

- CPU и Memory usage процесса
- Количество потоков
- Активные сетевые подключения
- Статистику приложения (WebSocket сессии, пользователи)
- Открытые file descriptors

Для однократного вывода:

```bash
./monitor_server.sh --once
```

### Вариант 2: API endpoint

```bash
curl http://localhost:3003/server-stats
```

### Вариант 3: Стандартные команды

```bash
# Найти PID процесса
pgrep -f WebSocketServer

# Мониторинг процесса
top -p $(pgrep -f WebSocketServer)

# Детальная информация
ps aux | grep WebSocketServer

# Сетевые подключения
ss -tnp | grep :3003

# Memory и CPU в реальном времени
watch -n 1 'ps -p $(pgrep -f WebSocketServer) -o %cpu,%mem,rss,vsz,cmd'
```

## 📈 Метрики для отслеживания

### CPU Usage

- **Норма:** < 10% при 100 соединениях
- **Внимание:** > 50% - оптимизация нужна
- **Критично:** > 80% - сервер перегружен

### Memory Usage

- **Норма:** ~100 KB на соединение
- **Внимание:** > 200 KB на соединение
- **Критично:** Постоянный рост (утечка памяти)

### Network Connections

- **Established:** Должно равняться количеству активных сессий
- **TIME_WAIT:** Закрытые соединения (должны исчезать)

### Database Sessions

- **Active Sessions:** Не истекшие сессии
- **Session Hit Rate:** % валидных сессий (должно быть 100% при тестах)

## 🔧 Функции страницы нагрузочного тестирования

### Создание сессий

- Автоматически регистрирует пользователя если не существует
- Создает N сессий последовательно
- Автоматически подключает WebSocket для каждой
- Progress bar показывает прогресс

### Мониторинг

- Обновление каждые 2 секунды
- Статистика сервера с /server-stats endpoint
- Локальная статистика (сообщения, ошибки)

### Broadcast сообщений

- Отправляет одно сообщение всем активным сессиям
- Показывает сколько сообщений отправлено
- Обновляет счетчик отправленных сообщений

### Stop All

- Закрывает все WebSocket соединения
- Очищает статистику
- Сбрасывает UI

## 🐛 Troubleshooting

### Проблема: Не все соединения подключаются

**Решение:**

1. Проверьте лимиты: `ulimit -n`
2. Увеличьте если нужно: `ulimit -n 10000`
3. Проверьте логи сервера: `sudo journalctl -u vapor-chat -f`

### Проблема: Соединения разрываются через 20 секунд

**Решение:**

- Проверьте что HTML страница отвечает на ping с pong
- Откройте консоль браузера (F12) и проверьте ошибки

### Проблема: High CPU usage

**Возможные причины:**

1. Слишком много ping/pong сообщений
2. Большой объем broadcast сообщений
3. Проверьте таймеры в коде

### Проблема: Memory leak

**Диагностика:**

```bash
# Мониторинг памяти в реальном времени
watch -n 1 'ps -p $(pgrep -f WebSocketServer) -o rss,vsz'

# Если память постоянно растет:
# 1. Проверьте что WebSocket соединения правильно закрываются
# 2. Проверьте cleanup timer для сессий
# 3. Посмотрите логи на утечки
```

## 📝 Примеры тестов

### Быстрый функциональный тест

```bash
# Terminal 1: Запустить мониторинг
./monitor_server.sh

# Terminal 2: Создать 10 подключений через curl
for i in {1..10}; do
    SESSION=$(curl -s -X POST http://localhost:3003/login \
        -H "Content-Type: application/json" \
        -d '{"username":"loadtest","password":"test123"}' | jq -r .sessionId)
    echo "Session $i: $SESSION"
done

# Проверить в мониторе: должно быть 10 сессий в БД
```

### WebSocket соединения через wscat

```bash
# Установить wscat если нет
npm install -g wscat

# Получить sessionId
SESSION=$(curl -s -X POST http://localhost:3003/login \
    -H "Content-Type: application/json" \
    -d '{"username":"loadtest","password":"test123"}' | jq -r .sessionId)

# Подключиться
wscat -c "ws://localhost:3003/chat?sessionId=$SESSION"

# В интерактивном режиме отправить:
{"type":"subscribe","messageType":"all_messages"}
{"type":"log","content":"Test message"}
```

## 🎯 Рекомендации

1. **Начинайте с малого:** Сначала 10, потом 100, потом больше
2. **Мониторьте:** Всегда следите за ресурсами сервера
3. **Проверяйте логи:** `sudo journalctl -u vapor-chat -f`
4. **Cleanup после тестов:** Нажимайте "Stop All" чтобы освободить ресурсы
5. **Database cleanup:** Периодически чистите старые сессии:
   ```bash
   sqlite3 db.sqlite "DELETE FROM sessions WHERE expires_at < $(date +%s)"
   ```
